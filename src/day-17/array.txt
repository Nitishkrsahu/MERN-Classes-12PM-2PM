                                    The "IF" statement:
- It is a decision making statement with various forms.
- It includes approach like
    a) Forward Jump
    b) Simple Decision
    c) Multiple Choices
    d) Multi Level Decisions

Forward Jump:
- It is a decision making approach where statements are executed only when condition evaluates to true.
- It will not provide any alternate process.

Syntax:
        if (boolean_expression)
        {
          statements on true;
        }
                                       IF Statement
1.Forward Jump
2. Simple Decision
- It provides an alternative to execute when condition evaluates to false.
- Alternative process is defined using "else" clause.

Syntax:
    if (condition)
    {
      statements on true;
    }
    else
    {
      statements on false;
    }

3. Multi Level Decisions:
- It is the process of verifying the next level only when the first evaluates to true.
- It allows to handle multiple conditions one after another.
- It uses forward jump and allows to check any specific issue in given set of conditions.

Syntax: [Ladder IF]
     if (condition1)
     {
          if(condition2) {
          statements when both conditions evaluates to true;
          }
         else {
         statement when condition2 is false;
         }
     }
     else
     {
       statements on condition1 false;
     }
                                   4. Multiple Choices
- It is the process verifying other conditions even when the given evaluates to false.
- You can provide multiple choices to handle same task.
- It will be clear in handling specific choice of statements on true and verify others if given is not true.

Syntax:
    if (condition1)
    {
      statements on condition1 true;
    }
    else if(condition2)
    {
      statements on condition2 true;
    }
    else
    {
     statements when all conditions are false;
    }
Note: Multiple decisions are not good when there are several choices to verify.
      It can't jump directly to the matching block of statements. It have to execute
      all conditions up to the matching. It is slow when there are many decisions.

Switch Selector
- Switch is used to interrupt the flow of electrons.
- A switch selector is used to handle multiple decisions and execute exactly the block that is matching with given condition.

Syntax:
    switch (value | condition)
    {
       case  value|condition:
        statements;
        jump;
      default:
        statements;
        jump;
    }

- Default executes the statements when value is not matching with any given case.
- Jump statement can be any one of the following
    a) break
    b) return
FAQ's:
1. Can we define switch without default block?
A. Yes. It is optional.

2. Can we define default above or between cases?
A. Yes. There is no order dependency.

3. Why to remove default?
A. To configure forward jump approach.

4. Can we define case without jump statement?
A. Yes. It will continue to the next case and stops at next occurrence of "jump".
5. Can we define "return" as jump statement?
A. Yes.

6. What is difference between break and return?
A. Break ends the current block but continues the execution. It keeps the compiling
     process ON.
    "Return" terminates the compiling process, and any statement defined after return
     is not reachable to compiler.
7. What can be value type or expression type for case & switch?
A. It is recommended to have a number, string or Boolean.
8. How to execute same block for multiple cases?
A. By writing multiple cases for same block.

        case 1:
        case 2:
        case 3:
            statements to execute;
            jump;


9. How to execute same block for a ranges of values that match given case?
A. You can check the range by using Boolean expression.
     If cases are using a Boolean expression, then the switch must be always true.

Syntax:
    switch(true)
    {
       case boolean_expression:
        statements;
        jump;
    }

- Arrays can reduce complexity by storing multiple values under one reference name.

- However few technologies restrict array for same type of values.

- Array refers to a type of arrangement of elements in a sequential order, where we have semantic purpose of arrangement. However we can access in random.
