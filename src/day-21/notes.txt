                                     JavaScript Fetch API
- JavaScript ES5+ version provides a Fetch API.
- It is a JavaScript promise used to handle Ajax request.
- Promise is by default async.
- It is simple and faster in handling AJAX request.


Syntax:
    fetch("url")
    .then(function(response){
         return  response.json();
    })
    .then(function(data_json){
        // use data
    })
    .catch(function(error){
        // report the error
    })
    .finally(function(){
        // executes always
    })

- "fetch()" is a window method.
- "then()" executes on success [ resolve ].
- "catch()" executes on failure [ reject ].
- "finally()" executes always.

                                                      JavaScript Map Type
- It is a key and value collection.
- Key can be any type.
- It provides various implicit methods for manipulation.
- It is faster when compared to object.
- The only issue with map is, it is structure less.

Syntax:
    var data = new Map();

    data.set(key, value)        store value
    data.get(key)                access value
    data.has(key)                verify key
    data.delete(key)            remove value using key
    data.clear()                removes all
    data.keys()                returns all keys
    data.values()                returns all values
    data.entries()                returns all keys and values
    data.size                    returns the count of keys

                                                                    Timer Events

1. setTimeout()
- A computer performs given set of task at high processing speed.
- It makes the processor busy and leads to hang up.
- You can delay the task by specific duration.
- The process of executing tasks one after another at high speed is known as "bouncing".
- It requires to implement "debounce".
- Debounce loads task into memory and keeps it inactive for a specific duration.

Syntax:
    setTimeout(function(){ }, interval)

2. clearTimeout()
- It is used to remove the task from memory before it is released into process.
- It requires a memory reference where the given task is stored.

Syntax:
       const  thread =  setTimeout(function(){ }, interval);

       clearTimeout(thread);
Note: Timeout executes task only once, as it removes from memory and send to the process. If you want to execute again, then load into memory again.
